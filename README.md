# CareCore: Smart Healthcare Management System 🏥

## 📌 Academic Context
This repository contains the complete **System Analysis and Design (SAD)** documentation for "CareCore," a cloud-based Software as a Service (SaaS) designed to transition traditional private clinics into 100% paperless environments. 
* **Developer:** Yousef Elnaggar (Solo Developer)
* **Academic Year:** 2025-2026

*Note: This repository currently serves as the architectural and software engineering blueprint (Documentation Phase) prior to the coding implementation.*

## ⚙️ Software Development Life Cycle (SDLC)
The project strictly adheres to the **Waterfall Methodology**, ensuring highly disciplined and sequential engineering phases. The development process spans 84 days (12 weeks) and encompasses:
1. **Planning & Feasibility:** Economic (ROI calculation), Technical, and Organisational feasibility studies.
2. **System Analysis:** Requirements elicitation via medical staff interviews and patient digital surveys (visualised via charts), defining strict Functional and Nonfunctional (GDPR-compliant, AES-256 encryption) requirements.
3. **System & Database Design:** Structural data modelling and process flow mapping.
4. **Implementation Planning:** System acquisition strategy and mathematical Alternative Matrix (selecting Flutter natively compiled framework).

## 📐 System Architecture & Modelling
To ensure a robust and scalable backend infrastructure, the system logic was rigorously mapped using the following engineering standards:
* **Process Modelling:** Context Diagram (DFD Level 0) and Data Flow Diagram (DFD Level 1).
* **Structural Modelling:** Entity-Relationship Diagram (ERD) defining Primary/Foreign keys and 1:M / 1:1 referential integrities.
* **Unified Modelling Language (UML):** Comprehensive Use Case Diagrams, Class Diagrams, Sequence Diagrams, and Activity Diagrams to map dynamic decision-making pathways and actor permissions (Patient, Doctor, Secretary).

## 📱 User Interface (UI) Design
The documentation includes meticulously designed UI mockups driven by usability constraints:
* **Patient Mobile Application:** Designed for absolute simplicity, featuring interactive calendars and one-tap booking to prevent schedule conflicts.
* **Clinic Web Dashboard:** A centralised, distraction-free control hub for medical staff to manage real-time queues and securely generate Electronic Health Records (EHR) and QR-coded digital prescriptions.
